# Security Policy

This project is in the foundation phase and has not published a production-ready release.

Do not include private data or detailed system information in public Issues. Use GitHub private reporting when available. Otherwise open a short Issue requesting a private communication channel.

A report should identify the affected commit, environment, reproduction steps, observed impact, and a sanitized example.

Changes involving local files, benchmark inputs, exported reports, logs, resource observation, or process control require careful review and clear user-facing behavior.

Validated reports will be reviewed privately where practical. Public notes should identify affected versions and corrections without exposing user information.
